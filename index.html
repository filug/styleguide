Google C++ Style Guide fork

Google made very helpful Google C++ Style Guide. In addition small (small but still power full) cpplint.py Python script was prepared by them.

CMake support

Original styleguide repository from Google doesn't contain any support for CMake build system.

Here required CMake module is available and it's quite easy to integrate Google C++ Style Guide with any CMake based project.

Setup

  1. Clone git@github.com:filug/styleguide.git repository somewhere inside your project,
  2. Include cpplint.cmake file in your root CMakeLists.txt file, for example: include(3rdParty/styleguide/cmake/cpplint.cmake)
  3. Configure which project directories should be checked by cpplint.py.

Usage

You can run Source code style verification via make lint command. In addition source code style verification will be also checked when target all will be built.

Output

$ make
[100%] cpplint.py: Checking source code style
Done processing /home/filug/projects/test-prj/src/dir1/file1.c
Done processing /home/filug/projects/test-prj/src/dir1/file1.h
Done processing /home/filug/projects/test-prj/src/common.h
Total errors found: 0

CTest

cpplint.cmake module is also using add_test command to add lint to the CTest framework.

Any time when make test command is started cpplint.py is checking source code.

Example:

$ make test
Running tests...
Test project /home/filug/projects/uleapp/build
    Start 1: lint.src
